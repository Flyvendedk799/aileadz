# aileadz — HR Value-Improvement Plan

_Generated by a multi-agent scaffold (5 explorers -> synthesize -> adversarial critique -> finalize). Every initiative is grounded in real files/routes; statuses verified as wired / dormant / dead._

## Thesis

aileadz is ~85-90% through its roadmap with a broad, genuinely functional HR control plane and a strong token-bound viz layer, so the highest-value moves are mostly harvesting what is already built: unify the split-brain navigation, render data the routes already compute, send emails whose templates already exist, fix the skill scale that understates competence, and wrap dormant predictive/calendar/compliance/analytics infra into proactive value. Critically, three of those harvests touch people-level analytics that are NOT actually k-anon-suppressed today (predictive churn returns named employees; funnel and attributed-revenue have no suppression), so the plan adds the missing k-anon floors and the last-learning-activity accuracy fix as explicit prerequisites before exposing them — and keeps the HR AI panel off shared fm_base to avoid leaking it to admin/vendor/public surfaces. The one strategic net-new bet is an outcome-measurement loop (skill-history + capture pipeline) that proves training actually closes gaps; sequenced cheap-coherence-and-trust-first, this raises perceived value fast while strictly respecting the sales-led, GDPR/k-anon, and data-viz constraints.

## Phasing (sequenced by impact / effort)

### Phase 1 — Coherence, trust quick-fix & quick activations (days, not weeks)

Make the already-built product feel whole, fix the one hard-constraint violation, land the cheapest trust fix (skill scale), and light up dormant channels that need no backend work. Highest impact per unit effort, no people-level k-anon exposure introduced.

- Unify HR navigation on the canonical _hr_subnav.html (kill the ~10 inline subnavs)
- Reframe seat/trial copy from self-serve 'Opgradér dit abonnement' to sales-led account-manager contact
- Send the already-built approval-needed and budget-overrun emails to managers
- Surface the existing .ics add-to-calendar action and an in-app subscribable course feed
- Fix the 1-4 vs 1-5 skill-level scale across data and viz (trust prerequisite)
- Add a k-anon-safe compliance-rate visual (donut + most-at-risk requirement bars)
- Include the embedded AI assistant on each HR page (per-page include, NOT fm_base) and token-fix the chart tooltip

### Phase 2 — Trustworthy analytics, accuracy & buyer-facing value

Fix the analytics that systematically mislead (skill scale already done in P1; now the at-risk signal), recover compute already thrown away, surface renewal-justifying numbers to the buyer WITH a real k-anon floor, and harden the AI money path.

- Unify a 'last learning activity' signal so at-risk/churn detection is trustworthy
- Render the data learning_analytics already computes (with correct labels) and add it to the nav
- Expose the tenant-scoped funnel / retention / AI-attributed-revenue to HR, adding a small-tenant suppression floor
- Add a nightly company_analytics rollup job (cheap perf + trend data)
- Wire grounding + untrusted-text fencing into the HR agent path

### Phase 3 — Proactive loops & advisor depth

Close the proactive automation loops (compliance, deadlines) on infra that is registered but idle, and deepen the advisor toward comparison, action and accurate early-warning conversations — only after the accuracy and k-anon prerequisites land.

- Activate the deferred compliance_recheck job into proactive recertification reminders
- Add a learning-deadline reminder job to the scheduler
- Add a get_workforce_risk advisor tool with an explicit aggregate/k-anon mode
- Add confirm-gated write tools (set_skill_target, create_compliance_requirement) so the advisor can close the loop
- Add an hr_compare_cohorts advisor tool with net-new per-cohort k-anon
- Expand benchmarking with skill-gap-closure, budget-utilization and engagement peer metrics

### Phase 4 — Strategic outcome measurement

Build the one genuine net-new capability that proves the core promise: evidence that training actually closes skill gaps, turning ROI from hypothetical to measured. Acknowledged XL because the entire capture pipeline is net-new.

- Skill-uplift loop: measure gap closure with a skill-history snapshot, not just spend

Apply consistently to the merge in get_skill_gap_analysis, the dashboard skill tiles, and skill_gaps.html copy/radar max/sliders. Add a one-line comment documenting the canonical scale.

**Value added.** Restores trust in the single best visualization on the surface. **Verdict.** Solid harvest-what-is-built plan with correct Phase-1 instincts, but it overstates k-anon safety on three AI/analytics initiatives (get_workforce_risk and funnel/attributed-revenue are NOT actually suppressed and expose named/identifiable data), proposes a defect-prone AI-panel-in-fm_base, ships predictive risk on a noisy chatbot-only proxy, and omits the accuracy fix, write-loop tools, and benchmarking expansion the findings support — fix the k-anon claims and demote get_workforce_risk before approving.
